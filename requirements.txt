# Two-tier pinning strategy:
#
# Tier 1 — exact pin. The native appsi Gurobi APIs we use
# (config.load_solution=False; res.best_feasible_objective;
# res.solution_loader.load_vars()) have shifted across minor releases,
# and the legacy appsi wrapper crashes on GDP-transformed models on
# older Pyomo. A hard pin guarantees identical solver behavior on every
# rebuild. Bump deliberately + test before deploy.
pyomo==6.10.0
gurobipy~=13.0
#
# Tier 2 — compatible release. Stable surface APIs at our usage level
# (sliders, charts), so accept patch upgrades for security and bug fixes.
# pandas stays on 2.x: Streamlit 1.52 caps pandas <3, so pandas~=3.0 makes
# the image's pip resolve impossible. The app uses no st.dataframe/.style, so
# no jinja2 (Styler) pin is needed.
streamlit~=1.52.0
pandas~=2.0
altair~=5.0.0
